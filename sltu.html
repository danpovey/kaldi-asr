Information for participants

The current version of the tutorial slides. Updated on 2016-05-10.

Logging onto a machine

Using the provided SSH private key, login on a assigned machine. Using the command line SSH (linux/macosX), this can be done using the following command: 
ssh -i sltu-public.pem.txt ubuntu@<machine-name>
where <machine-name> is the address of the machine with the same number you've been assigned. Please note that after downloading, you might need to change the access rights to the file sltu-public.pem.txt to allow only the current user to read the file (it's a security precaution built-in into OpenSSH clients). This can be done using the following command 
chmod 600 sltu-public.pem.txt

Running the recipe

Once you've log in, run the command screen. That command ensures that even if the connection to the machine is dropped, the scripts will still keep running, so you won't have to run everything from scratch.

The recipe is in ~/kaldi/egs/iban/s5. Everything should be set up correctly for you, so that you can go ahead and run the script run.sh. We will use tee to keep the console output in the log file for a future reference. 

~/kaldi/egs/iban/s5 $ ./run.sh 2>&1 | tee run.log
It will take a two or three hours to finish. After the script finishes, you can run another script, providing training procedure for TDNN acoustic models. That script is ./local/nnet3/run_tdnn.sh. It is inteded to be run from the same directory in which the script run.sh lies, i.e. should be executed as 
~/kaldi/egs/iban/s5 $ ./local/nnet3/run_tdnn.sh 2>&1 | tee run-tdnn.log

List of machines

The machines are now (as of 2016-05-10) offline. The Iban recipe is include into the Kaldi standard egs. The data are publicly available (see http://www.openslr.org/24/). You can train the system using your own local machine/cluster.

We have published two AWS AMI images "v4 IBAN recipe/Ubuntu 14.04LTS" (AMI-ID ami-7b688916) and "v5 IBAN recipe/Ubuntu 14.04LTS" (AMI-ID ami-6945a804). The difference between the two is tht "v4" has a complete setup, but does not contain the files created during training. The "v5" contains fully trained and decoded system.
